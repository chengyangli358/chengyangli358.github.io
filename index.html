flex(弹性)布局： 通过给父盒子添加flex属性(display:flex)，控制子盒子的位置和排列方式

当我们为父盒子设为flex布局以后，子元素的float、clear和vertical-align属性将失效

常见的父项属性：

     ![](/123.png)

flex-direction(主轴):row(默认 从左到右)、row-reverse(从右到左)、column(从上到下)、column-reverse(从下到上)

justify-content(排列方式，根据主轴方向)：flex-start(默认值，从头部开始)、flex-end(从尾部开始，顺序不变)、center(在主轴居中对齐)、space-around(评分剩余空间)、space-between(先两边贴边，再平分剩余空间)

flex-warp(是否换行，在flex中，子元素不会自动换行，一行放不开的时候会压缩子盒子的大小) warp、nowarp

align-items(侧轴排列方式 单行) flex-start(默认值 从上往下)、flex-end(从下往上)、center(挤在一起居中/垂直居中)、stretch(拉伸 盒子不能给高度)

align-content(多行/子项出现换行的情况) flex-start(默认值，从头部开始)、flex-end(从尾部开始，顺序不变)、center(在主轴居中对齐)、space-around(评分剩余空间)、space-between(先两边贴边，再平分剩余空间)、stretch(子项元素高度平分父元素高度)

子项属性：

flex：定义子项目分配剩余空间，用flex来表示占多少份数 默认为零

align-self：控制子项自己在侧轴的对齐方式

order：属性定义项目的排列顺序 跟数字 越小越往前，默认为零

移动布局：

视口(viewport)： 浏览器显示页面内容的屏幕区域，可分为布局视口、视觉视口、理想视口

1.布局视口(layout viewport) ：浏览器默认设置的布局视口，用于解决早期pc端页面在手机上显示的问题

2.视觉视口(visual viewport) ：手机浏览器显示的部分

3.理想视口(ideal viewport)  ：设备有多宽，布局视口就有多宽。 需要手动添加meta视口标签

meta视口标签：
标准的viewport设置

key：PC端页面1px是等于一个物理像素(屏幕显示的最小颗粒 就是分辨率)的，移动端一个px能显示的物理像素点 的个数，称为物理像素比或屏幕像素比

倍图：提高倍数提高图片质量，避免图片在手机端放大模糊

原理：准备的图片比我们实际需要的大小大手机端放大的倍数，然后缩小图，这样在手机端显示的就是图片原本 的清晰度

cover：把背景图像扩展至足够大，使背景图像完全覆盖背景区域(等比缩放，填满，会超出区域)

contain：把图像扩展至最大尺寸，以使其宽度和高度完全适应内容区域(等比缩放，宽或高一个铺 满就不再拉伸，会有空白区域)

定位(position)： 定位=定位模式+边偏移

在CSS中，通过top、bottom、left、right属性定义边偏移

定位模式：语法： 选择器{ position：属性值；}

属性值：static：静态定位 (默认的定位，没有边偏移，一般不使用)

    relative：相对定位 (相对自身原来的位置偏移，原来占有的区域依然占有)

absolute：绝对定位 (以带有定位的父级元素移动位置，如果父级没有定位，则 以浏览器为标准，完全脱标，不占位置)("子元素如果用绝对定位，父级要用相对定位") 也会将display改为行内块

堆叠顺序：可以用z-index：数值；数值越大盒子就放在上面，没有单位，只适用于定位

浮动元素和绝对定位都不会发生外边距合并的问题

浮动(float)
为了使块级元素可以在一行显示(行内块会有空隙 浮动没有)

属性：none、left、right

特性：1.脱标(脱离标准流，浮动在标准流的上面)
2.盒子浮起来之后，原来的位置就给了下面的盒子
3.浮动会改变display属性，变为类似行内块

为了避免浮动脱标影响下面的标准流元素，所以需要给浮动的元素添加一个标准流的父亲

清除浮动：因为很多情况下，不方便给父元素高度，以达到子元素有多少内容就显示多少内容
但是因为浮动不占高度，所以父元素的高度为零，下面的标准流盒子就会挤上来，影响页面布局
本质：清除浮动之后，父级就会自动检测高度，不会影响下面的布局
方法：语法：clear：{} 属性值：left：不允许左侧有浮动元素(清除左侧浮动的影响)
right：不允许右侧有浮动元素(清除右侧浮动的影响)
both：同时清除两侧浮动的影响(一般就使用这个)
1.额外标签法(隔墙法)：在最后浮动元素的末尾添加一个空标签，样式就是clear both
通俗易懂，但是添加了许多无意义的标签，结构化较差
2.父级元素添加overflow：hidden|auto|scroll 都可以解决
方便，但是会隐藏溢出的内容/生成滚动条 影响美观
3.使用：after 伪元素消除浮动
添加 .clearfix(父元素的class，可自己取名字)：after {
content: “”;
display: block;
height: 0;
visibility:hidden;
clear: both;
} 高版本
.clearfix {
*zoom: 1;
} 低版本浏览器
相当于自动生成了新的after标签添在后面，相当于升级版的额外标签法
4.使用双伪类
.clearfix：before，
.clearfix：after {
content： “”;
display： table；
}
.clearfix：after {
clear：both；
}
.clearfix {
*zoom: 1;
}

CSS盒子模型(box model)：
1.边框border：border-width：边框粗细

border-color：边框颜色

border-style：边框样式

   样式：none(默认无)、solid(单实线)、dashed(虚线)、dotted(点线)等

   可综合写，排列无顺序 border：粗细 样式 颜色

   可分别对上(top)下(bottom)左(left)右(right)的边框规定样式

   边框折叠：border-collapse：collapse

2.内边距padding：添加会使得盒子变大,(如果这个盒子没有指定宽度，则不会撑开盒子)

也有四个方位(padding-top/bottom/left/right)

简写：一个值：上下左右都是相同像素的内边距

      两个值：上下第一个值，左右一个值
      三个值：上、左右、下
      四个值：上、右、下、左(顺时针)

3.外边距margin：实现块级元素水平居中：1.有宽度 2.左右外边距设置为auto

清除元素默认内外边距：*{margin：0；padding：0} 基本页面都会用到
行内尽量只设置左右边距，不设置上下的(浏览器不兼容)

上下外边距的合并(塌陷)：当上下两个相邻的块级元素相遇时，如果上面的元素有下边距margin-bottom，下面的元 素有上边距margin-top，则他们之间的垂直间距不是上下边距的和而是取两个值中较大的那一个，为避免这种情况， 尽量只给一个盒子添加margin的值

嵌套块元素的边距合并(塌陷)：对于两个嵌套关系的块元素，如果父元素没有上内边距及边框，则父元素的上边距会 与子元素的上外边距发生合并，合并后的外边距为两者中的较大者。解决方案：可以为父元素定义上边框/内边距、 可以为父元素添加overflow：hidden

为了盒子的稳定性：width>padding>margin

标签显示模式(display)：

2.行内元素span(inlink-level)：一行可以放多个，高宽设置皆无效，默认宽度看内容，里面只放文字/行 (span),链接里面不再放(链接)

3.行内块元素：(img、input、td)：可以设置宽高，一行可以放多个，相隔之间有空隙，宽高边距可设置

转换：行内元素转块：display：block
块转行内：display：inline
转行内快：display：inline-block

行高=高度(块) 文字水平居中
行高>高度(块) 文字偏下
行高<高度(块) 文字偏上

##Css 层叠样式表：布局和外观

书写位置： 1.行内(内联)样式表：通过style属性直接写

2.内部样式表，style标签中通过选择器规定样式，只作用于当前页面

3.外部样式表：称嵌入式。
     将所有的样式放在一个或者多个以.CSS为扩展名的外部样式表文件中，通过link标签将外部样式表文件链接到HTML文档中》

link为单标签，需要放在head头部标签内，并且指定link标签的三个属性： 1.rel(定义当前文档与被链接文档之间的关系：stylesheet，表示链接的是一个样式表文件)、

2.type(定义所链接文档的类型：text/CSS，表示链接的外部文件为CSS样式表，可省略)、

3.href(定义所链接外部样式表文件的URL，可以是相对路径也可以是绝对路径)  可以实现样式共享

选择器： 1.标签(元素)选择器：通过标签规定样式

2.类选择器：语法：.类名 {样式} 给标签规定类(class)调用 特殊用法：多类名(class="one two")

3.id选择器：语法：#id {样式} 元素的id是唯一的，不能重复

4.通配符选择器：语法：* {样式} 匹配页面所有的标签

字体(font)样式： 综合写法：font：风格 粗细 大小 字体 (顺序是固定的，可省略，不可更改，字号字体必须同时出现)

1.font-size(字体大小)

2.font-family(字体) 同时指定多个字体用逗号分开，计算机遍历到计算机有的字体使用、Unicode字体(避免版本和浏览器的不同导致乱码) 例：宋体：\5B8B\4F53

3.font-weight(字体粗细)100-900，normal(默认约等于400)，bold(加粗 约等于700)

4.font-style(字体风格)，normal(默认)，Italic(斜体)

5.text-align(字体对齐方式)，center(居中)，left(左对齐)，right(右对齐)

6.line-height(行间距)

7.text-indent(首行缩进)，一般使用em(1em=1汉字)为单位

8.text-decoration(文本的装饰)：none(无装饰)、blink(闪烁)、underline(下划线)、line-through(贯穿线)、overline(上划线)
